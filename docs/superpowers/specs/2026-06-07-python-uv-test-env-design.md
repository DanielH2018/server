# Python dev/test environment via uv — design

**Date:** 2026-06-07
**Status:** Approved (pending spec review)
**Goal:** Give the repo a single, reproducible Python dev/test environment managed by
[uv](https://docs.astral.sh/uv/), with one source of truth for test dependencies, a pytest
config that runs every repo-owned suite (closing the `scripts/` CI gap) and excludes the
vendored Ansible collections, and a single documented command to run the tests locally and
in CI.

## Motivation / current state

- **No Python manifest.** Test/dev deps (`pytest`, `ansible-core`, `pyyaml`, `jinja2`) exist
  only as inline `additional_dependencies` lists inside `prek.toml` hooks. No `pyproject.toml`,
  no `requirements*.txt`.
- **No one-command local run.** System `python3` (3.12) has neither pytest nor ansible
  (ansible-core is a pipx install). Tests run only inside prek's auto-provisioned isolated
  envs, or a hand-rolled venv.
- **No pytest config.** No `testpaths`/`pyproject.toml`, so a bare `pytest` from the root
  recurses into the third-party `ansible/collections/**` tests. The prek hook works around
  this by passing explicit path `args`.
- **CI gap.** `scripts/test_smoke_extract.py` exists but is *not* in the prek pytest hook's
  `args`/`files`, so neither prek nor CI ever runs it.
- **Inconsistent run conventions.** Four repo-owned suites with three documented invocations
  (`python3 -m pytest …`, `python3 -m unittest test_check`, standalone `python3 …`).

## Repo-owned test suites (the only ones in scope)

| Suite | Module under test | Purpose |
|-------|-------------------|---------|
| `ansible/tests/test_toposort.py` | `ansible/filter_plugins/toposort.py` | deploy ordering/scope filters |
| `ansible/roles/containers/monitor-bridge/files/test_check.py` | `check.py` | Kopia backup-age / RFC3339 parsing |
| `.claude/hooks/test_auto_approve_readonly.py` | `auto-approve-readonly.py` | read-only Bash classifier (permission boundary) |
| `scripts/test_smoke_extract.py` | `smoke_extract.py` | changed-image-ref extraction |

`ansible/collections/**` tests are third-party (vendored) and explicitly out of scope.

## Approach

**uv drives the Python tooling end to end (chosen over uv-as-local-convenience-only).**
The prek `pytest` and `validate-compose-templates` hooks call `uv run …` instead of
self-provisioning, so dependency versions live in exactly one place: `pyproject.toml`'s
`dev` dependency group. uv becomes a prerequisite for `prek run` (CI installs it via
`setup-uv`; developers install it once). This is the accepted cost of true
single-source-of-truth.

## Changes

### New: `pyproject.toml` (repo root)

```toml
[project]
name = "server-homelab"
version = "0.0.0"
requires-python = ">=3.12"

[tool.uv]
package = false                      # virtual project: manage deps/lock, do not build the repo

[dependency-groups]
dev = [                              # uv includes a group named `dev` by default for `uv run`/`uv sync`
  "pytest>=8",
  "ansible-core>=2.18",             # test_toposort imports ansible.errors
  "pyyaml",                         # validate_compose_templates
  "jinja2>=3",                      # validate_compose_templates
]

[tool.pytest.ini_options]
testpaths = [                        # single source of truth for WHAT runs; restricts collection
  "ansible/tests",                   # away from vendored ansible/collections
  "ansible/roles/containers/monitor-bridge/files",
  ".claude/hooks",
  "scripts",                         # closes the smoke_extract CI gap
]
addopts = "-q"
pythonpath = ["ansible/filter_plugins"]   # test_toposort can import toposort (replaces conftest.py)
```

Notes:
- The `dev` group name is deliberate: uv treats it as the default group, so the prek/CI entry
  is just `uv run python -m pytest` — no `--group` flag.
- `testpaths` makes the per-suite paths a single source of truth consumed by both local
  `uv run pytest` and the prek hook. The other three suites import their module-under-test from
  their own directory (pytest's prepend import mode handles that); only `test_toposort` needs the
  `pythonpath` entry because its module lives in a different directory.

### New: `uv.lock` (repo root)

Generated by `uv lock`; committed for reproducibility.

### Edit: `.gitignore`

The file is allowlist-style (`/*` denies root, `!/path` re-includes). New root files must be
explicitly allowlisted:

```gitignore
!/pyproject.toml
!/uv.lock
__pycache__/
.pytest_cache/
```

`.venv/` needs no entry — it is already denied by the root `/*` rule.

### Edit: `prek.toml`

**`pytest` hook** — switch from prek's isolated python env to uv:
- `language = "system"`
- `entry = "uv run python -m pytest"`
- remove `additional_dependencies` (now from `pyproject.toml`)
- remove `args` (pytest reads `testpaths`)
- extend `files` regex to add `scripts/.*\.py` and `pyproject\.toml`
- update the hook comment to point at `pyproject.toml`'s `dev` group

**`validate-compose-templates` hook** — same switch:
- `language = "system"`
- `entry = "uv run python scripts/validate_compose_templates.py"`
- remove `additional_dependencies`
- `files` / `pass_filenames` unchanged

### Edit: `.github/workflows/ci.yml`

Add a uv install step before `prek run --all-files`:

```yaml
      - name: Install uv
        uses: astral-sh/setup-uv@v5
```

`uv run` auto-syncs from `uv.lock` on first invocation, so no explicit `uv sync` step is
needed. `setup-python` (3.12) is retained.

### Consolidation

- **Delete `ansible/tests/conftest.py`** — its sys.path insertion is replaced by `pythonpath`
  in `pyproject.toml`.
- **Convert `ansible/roles/containers/monitor-bridge/files/test_check.py`** from
  `unittest.TestCase` + `unittest.mock` to plain pytest functions with `monkeypatch`,
  preserving every assertion, for stylistic consistency across all four suites.
- **Unify run-instruction docstrings** in all four test files to the single command
  `uv run pytest` (or `uv run pytest <path>` for a single suite).

## Trade-offs

- **uv is now required for `prek run`.** The `pytest` + `validate-compose-templates` hooks shell
  out to uv, so a machine without uv fails those hooks. Mitigated: CI installs uv via
  `setup-uv`; developers install uv once. This is the deliberate cost of removing the
  `additional_dependencies` duplication.
- **Those two hooks lose prek's per-hook isolation** and share uv's project `.venv`. Acceptable —
  they are repo-internal tooling, not third-party hooks.

## Verification

1. Install uv; `uv lock && uv sync`.
2. `uv run pytest` — all four suites pass, **including `scripts/`**; vendored
   `ansible/collections/**` is not collected.
3. `prek run pytest --all-files` and `prek run validate-compose-templates --all-files` pass via uv.
4. `prek run --all-files` is green end to end.

## Out of scope

- Converting third-party `ansible/collections/**` tests.
- Changing the runtime Python for Ansible (remains pipx-managed).
- Adding coverage tooling, type checking, or linters beyond what already exists.
